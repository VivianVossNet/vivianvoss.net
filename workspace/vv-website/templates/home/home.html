The Program in the Kernel

The second of three Fridays on a dependable server OS. eBPF lets you load a program the kernel runs in Ring 0; one static analyser, the verifier, stands guard, and about 68% of eBPF vulnerability classes are the memory-safety bugs it exists to stop. The kernel now ships unprivileged eBPF off by default, its own verdict. DTrace gives the same trace, built so the dangerous program cannot be formed. A verifier can have a bug; a missing capability cannot. Security Review ⊣ Boring on Purpose, 2 of 3.

The Workaround That Became Architecture

The third of the four breaks, the workaround mistaken for architecture. Node.js chose a single-threaded runtime in 2009, so to fill a machine you run many containers where a scheduler would use one, each carrying a whole userland to patch. Go and Rust reach every core from one binary; FreeBSD jails isolated at almost no cost in 2000. Multiplied, it is data centres of power at no gain in speed. Lean Software ⊣ How We Got Here, 4 of 18.

The Ones With the Paper

First of four portraits of the Unix family. The Open Group certificate is a pass to a niche, not a bond of family, and the niches end every way: HP-UX dead since 2025, Solaris frozen yet forked free as illumos, AIX still built for POWER11, z/OS a Unix surface over a mainframe, and macOS really two systems over the open Darwin core. The Last Ten Per Cent

The one who builds everything to the maximum and calls it perfectionism is its opposite. Perfect is not the maximum, it is the fit. The last ten per cent costs more than the ninety before it, and what nine in ten never see is not the fit but surplus, paid in the making, the explaining, and the maker's nerves. Good work comes from reduction: the demoscene, KISS, lean. IT Philosophy ⊣ Standalone.

The Surface You Never Added

First of three Fridays on what makes a server OS dependable. In 2023 Google switched io_uring off, fleet-wide. The reason is a number: about six in ten Linux kernel exploits in Google's 2022 bounty came through that one subsystem. io_uring is the fifth I/O interface Linux accreted, moving async work into the kernel, all of it attack surface. FreeBSD waits through kqueue, one primitive since 2000. The safest feature is the one you never add. Security Review ⊣ Boring on Purpose, 1 of 3.
