Nine Years in the Page Cache

On 29 April 2026 a researcher at Theori disclosed CVE-2026-31431, named Copy Fail: a Linux kernel flaw that lets an unprivileged local user become root in 732 bytes of Python, on every distribution shipping a kernel from the last nine years. It does not race. It does not crash. Three correct decisions, three different teams, fifteen years apart, meeting in the page cache: AF_ALG (2010), authencesn (2011), AEAD splice (2015), in-place optimisation (2017). The composition gives a deterministic four-byte primitive against the in-memory copy of any file the user can read. The classical target is /usr/bin/su; the disk is unchanged; the page is never marked dirty; Tripwire, AIDE and EDR host-integrity tools see nothing. The page cache is shared between containers and the host, so the corridor crosses container boundaries. On FreeBSD the equivalent does not arise in the same form: crypto via /dev/crypto; Capsicum (FreeBSD 9, 2012) narrows the corridor structurally; jails are a kernel-native primitive. The recommendation is not "patch and move on"; it is check the seams. Security Review ⊣ A four-byte write, three optimisations, and a kernel that forgot the layer beneath.
